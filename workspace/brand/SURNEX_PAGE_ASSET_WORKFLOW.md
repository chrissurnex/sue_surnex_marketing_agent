# Surnex Page Asset Workflow v1

Date: 2026-03-17
Purpose: define the process for generating real website page-support images in a consistent, non-chaotic way.

---

## 1. Scope

This workflow is for real page assets such as:
- keyword research page visuals
- AI visibility page visuals
- reporting page visuals
- backlink page visuals
- technical SEO page visuals

These are not broad style explorations.
These are production-oriented page images.

---

## 2. Inputs Required

For each page set gather:
- page/section reference image if available
- real product screenshots if relevant
- subject/benefit labels for each image
- approved master style references

---

## 3. Rules

### Rule 1
Page assets use the **Master Page Asset Prompt**.

### Rule 2
Page assets use the **No-Logo Rule** by default.

### Rule 3
Page assets in the same row/set must feel coordinated.

### Rule 4
One page set should have one dominant style anchor.
Do not let each image drift into its own mini-world.

### Rule 5
If one image in a set is clearly strongest, use it as the style anchor for the next regeneration of the full set.

---

## 4. Recommended Production Loop

### Step 1 — Generate first set
Generate 3 page-support images from the same master style.

### Step 2 — Pick the strongest image
Ask:
- which image feels most on-brand?
- which one balances clarity and quality best?
- which one should define the set style?

### Step 3 — Rebuild entire set around the strongest image
Do not just tweak one weak image.
Use the strongest one as the style anchor and regenerate the whole set consistently.

### Step 4 — Lock the set
Once the set is visually coherent, mark it approved and move to the next page.

---

## 5. Page-Asset Quality Standard

A good page asset should be:
- clean enough for a product page
- distinctive enough to feel premium
- related to the product/page subject
- consistent with the rest of the page set
- free of logos and text unless explicitly requested

---

## 6. Anti-Patterns

Do not:
- paste dashboards directly
- let one image become much busier than the others
- allow logos into feature/page-support images
- let prompts drift per image so far that the set breaks visually
- optimize for visual spectacle over page usefulness

---

## 7. Keyword Research Reset Rule

For the keyword research page specifically:
- use the strongest approved image in the set as the style anchor
- no logo on any of the 3 images
- rebuild weak images to match the best one
- prioritize clean premium page usefulness over artistic novelty
