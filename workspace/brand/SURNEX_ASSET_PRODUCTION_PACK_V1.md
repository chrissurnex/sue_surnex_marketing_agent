## 3. Asset Specs and Creative Briefs

## 3.1 Asset 1 — Homepage Hero Visual

### Job
Make visitors feel that Surnex is a serious, modern, operator-grade platform for AI search and SEO — not another vague AI tool.

### Message to support
- modern search is fragmented and noisy
- Surnex brings structure, visibility, and advantage
- built for agencies and marketers who need real leverage

### Visual direction
Preferred direction:
- dark tactical environment
- one dominant visual object or dashboard-led composition
- controlled lime routing/signal energy
- premium, clean, memorable

### Best visual modes to test
1. **Product-led hero**
   - dominant dashboard composition
   - realistic Surnex UI cues
   - layered panels
2. **Abstract signal hero**
   - no literal UI
   - routed signal pathways
   - more category-level storytelling
3. **Hybrid hero**
   - realistic product core with brand signal environment

### Format specs
- Primary: 16:9
- Secondary: 21:9 / ultra-wide variant for flexible hero use

### Success criteria
- looks premium at a glance
- clearly B2B/product/technical
- leaves room for headline copy
- does not feel generic AI wallpaper

### Prompt starting point
> Premium homepage hero for Surnex featuring a realistic dark analytics dashboard in a tactical brand environment, persistent left sidebar, KPI cards, charts and dense operational modules, framed by controlled lime routing paths and subtle atmospheric signal energy, high contrast, premium and serious, built for an AI search and SEO platform for agencies, optimized for 16:9, avoid neon overload, fantasy interface design, and generic AI startup gradients

## 3.2 Asset 2 — AI Visibility Feature Visual

### Job
Visually represent AI visibility, citations, answer presence, brand mentions, and prompt-level intelligence.

### Message to support
- understand where your brand appears in AI answers
- identify visibility gaps and mention opportunities
- monitor modern search surfaces, not just classic rankings

### Visual direction
Two valid lanes:
1. product-fragment based visual
2. abstract routed-answer system visual

### Recommended approach
Use a hybrid:
- dashboard fragments
- routed answer/citation paths
- structured answer nodes

- 4:3 or 1:1 for section flexibility
- optional 16:9 wide variant

### Success criteria
- feels like AI search intelligence, not chatbot consumer UX
- avoids generic “robot brain” visuals
- visually distinct from classic SEO charts alone

### Prompt starting point
> Feature visual for Surnex AI visibility, dark technical composition combining realistic dashboard fragments with structured answer-routing paths, citation nodes, mention indicators, and disciplined lime green highlights, premium B2B SaaS aesthetic, serious and precise, suitable for a website feature section, no consumer chatbot UI, no sci-fi robot imagery, no generic AI wallpaper

## 5. Production Workflow

## Step 1 — Choose fidelity mode
Use one of:
- Mode A = Pure Brand Abstract
- Mode B = Product-Inspired
- Mode C = Product-Matching

## Step 2 — Choose asset family
Pick exact use case:
- hero
- feature
- banner
- blog
- announcement
- ad

## Step 3 — Add composition modifier
Examples:
- layered product stack
- tight crop
- sparse wide composition
- centered square focal object

## Step 4 — Add format modifier
Examples:
- optimized for 16:9
- optimized for 1:1
- optimized for 4:5

## Step 5 — Run 3-variation generation sets
Do not generate 20 random images at once.
Run:
- Variation A = safer / cleaner
- Variation B = balanced / recommended
- Variation C = bolder / more energetic

## Step 6 — Score outputs against system
Use these criteria:
- feels like Surnex
- supports the intended message
- clean composition
- right level of product fidelity
- lime discipline
- no generic AI drift

---

## 6. Scoring Rubric

Rate each output 1–5 on:
- Brand fit
- Product truth
- Visual impact
- Clarity
- Reusability
- Thumbnail strength

Kill any output that scores low on brand fit or product truth.

## 8. What Not to Do

Do not:
- generate random one-off art without an asset brief
- ask the model for “something cool”
- let typography inside images become part of the plan
- overstuff visuals with green
- make product visuals too clean to resemble the actual app
- use abstract AI clichés instead of Surnex-specific logic
