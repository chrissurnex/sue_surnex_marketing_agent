## 1. System Goal

Build a reusable design operating system for Surnex that is:
- consistent
- fast to apply
- strong enough for production
- flexible enough for campaigns
- grounded enough to stay aligned with the actual product

This system is for repeated execution, not one-off art experiments.

### Core attributes
- precise
- technical

- high-contrast

## 3. Color System

## 3.1 Brand Accent Palette

| Token | Hex | Use |
|---|---:|---|
| `brand-lime-300` | `#bbf451` | Soft highlights, chart support, subtle glow edges |
| `brand-lime-400` | `#9ae600` | Hover/active highlights, brighter accents |
| `brand-lime-500` | `#7ccf00` | Primary brand accent |
| `brand-lime-600` | `#5ea500` | Darker accent, pressed state, secondary emphasis |

### Accent rules
- Use `brand-lime-500` as the default green.
- Use brighter/lighter greens sparingly.
- Prefer concentrated green moments over full green surfaces.
- Large blocks of lime should be rare and deliberate.

## 3.2 Neutral Palette

| Token | Hex | Use |
|---|---:|---|
| `neutral-950` | `#0a0a0a` | Base background / primary dark field |
| `neutral-900` | `#171717` | Primary surface / panel |
| `neutral-800` | `#262626` | Secondary surface |
| `neutral-700` | `#404040` | Borders, strokes, dividers |
| `neutral-400` | `#a1a1a1` | Muted text |
| `neutral-50` | `#fafafa` (working) | Primary light text |

## 3.3 Color Usage Ratios

Suggested visual balance for brand/marketing:
- **75–85%** dark neutrals
- **10–20%** white/light text and supporting contrast
- **3–8%** lime accents
- **0–5%** supporting data/chart colors when needed

## 3.4 Supporting Data Viz Colors

For charts/product-led assets, allow pragmatic supporting colors such as:
- violet/purple
- cyan/teal
- blue
- amber/orange
- red

Rule:
- these exist for data differentiation
- they do **not** replace the core brand accent

## 4. Typography System

## 4.1 Direction

Choose a modern sans-serif family or pairing that supports:
- strong display headers
- readable UI/body copy
- clean numerical data presentation
- tight, modern rhythm

## 4.2 Hierarchy

### Display / Hero
- bold
- compact
- high contrast
- sentence case or controlled title case
- minimal gimmicks

### Headings
- clean, assertive, direct
- no novelty styling
- should work equally well in website sections and social graphics

### Body
- readable, neutral, product-grade
- no excessive line length on dark backgrounds

### Data / UI text
- precise, small-size legibility matters
- must support dense charts, labels, tables, and captions

## 4.3 Typography Rules
- Prefer short, strong headlines.
- Use weight and scale more than color for hierarchy.
- Avoid excessive all caps; use only for labels/chips where useful.
- Keep line breaks intentional and forceful.
- Numbers should read cleanly and confidently.

---

## 5. Spacing and Grid

## 5.1 Grid logic
Surnex visuals should feel structured.

Use:
- clear columns
- modular content blocks
- consistent gutters
- repeatable spacing rhythm

## 5.2 Spacing behavior
- generous enough to feel premium
- not so spacious that the brand feels empty or weak
- allow more density in product-led visuals than in pure marketing compositions

## 5.3 Layout rhythm
Preferred layout character:
- headline block
- supporting subcopy
- single dominant visual
- supporting panels or cards
- strong alignment

Avoid:
- floating unanchored elements
- arbitrary misalignment
- scattered decorative fragments

---

## 6. Shape System

## 6.1 Core shapes
- rounded rectangles
- segmented bars
- routing lines
- nodes
- circular rings or arcs
- cropped directional panels

## 6.2 Radius guidance
Working brand radius direction:
- medium rounding for panels/cards
- slightly tighter radii for chips/buttons
- enough softness to feel modern, not enough to feel playful

## 6.3 Line behavior
Lines should feel:
- thin to medium
- precise
- directional
- purposeful
- suitable for signal, flow, and routing metaphors

---

## 7. Border, Shadow, and Glow Rules

## 7.1 Borders
- thin, subtle, low-drama
- useful for panel definition and chart structure
- often better than shadows on dark surfaces

## 7.2 Shadows
- minimal
- mostly used to separate layered compositions subtly
- avoid fluffy soft drop-shadow stacks

## 7.3 Glow
Glow is allowed, but only as a controlled accent.

Good glow use:
- edge emphasis
- path-line energy
- node highlights
- signal concentration
- hero focal points

Bad glow use:
- full-scene neon wash
- glowing every card edge
- glow as a substitute for composition

---

## 8. Background System

## 8.1 Approved background families

### A. Void + Signal
- near-black field
- sparse lime paths or nodes
- dramatic but clean

### B. Tactical Grid
- subtle grid or matrix structure
- dark panel feel
- ideal for site sections, decks, and blog covers

### C. Routed Network
- pathing, nodes, directional structure
- useful for campaign graphics and explanatory visuals

### D. Product-Derived Dashboard Layer
- cropped or stylized product panels
- realistic UI fragments
- ideal for product-led marketing

## 8.2 Background rules
- backgrounds must support content, not fight it
- if text loses legibility, background is wrong
- prefer sparse control over visual clutter

---

## 9. Iconography Direction

Icons should be:
- minimal
- clean
- thin or medium stroke
- modern
- aligned to product-grade UI language

Avoid:
- cartoon icons
- novelty icon packs
- excessively rounded playful icons
- 3D icons

---

## 10. Product Representation Rules

When representing the Surnex product in visuals:
- preserve the dark dashboard shell
- preserve left-nav context when possible
- preserve realistic chart density
- preserve modular card layouts
- keep chart colors credible
- avoid rebuilding the UI into fantasy art

Marketing can stylize the framing.
Marketing should not falsify the interface.

## 11. Asset Templates by Use Case

## 11.1 Website Hero
Structure:
- strong headline left or centered
- one dominant visual object
- optional secondary supporting panel
- dark field + signal energy
- product or abstract route depending on message

## 11.2 Feature Section Visual
Structure:
- one focused product/abstract visual per capability
- limited number of elements
- highly legible at medium size

## 11.3 Social Cover / Banner
Structure:
- strong contrast
- one simple visual idea
- clear safe zones
- minimal text if any
- highly recognizable from a distance

## 11.4 Blog Cover
Structure:
- template-based title zone
- one supporting visual family
- repeatable series logic
- should scale across dozens of posts

## 11.5 Ad Creative
Structure:
- strongest message first
- visual must support offer immediately
- less detail than the website
- controlled aggression allowed

## 11.6 Deck / Presentation Slide
Structure:
- dark base
- direct copy
- charts or panels only when useful
- should feel sharp, not overloaded

## 13. Content Density Rules

### Brand/marketing pages
- moderate density
- let the work breathe
- use strong hierarchy

### Product-led compositions
- moderate-high density
- enough detail to feel real
- avoid toy-dashboard simplification

### Social/ad graphics
- lower density
- one idea at a time
- optimize for speed of understanding

---

## 14. Photography / Illustration Guidance

At this stage, Surnex should not rely heavily on stock human photography.

Preferred visual emphasis:
- product UI
- abstract signal systems
- technical environments
- charts/panels/data forms

If people ever appear, they should be secondary, not the core brand language.

---

## 15. Review Checklist

Before approving any Surnex visual, ask:

1. Does this feel like Surnex, or like a random AI company?
2. Is the contrast strong enough?
3. Is the green used with discipline?
4. Is the structure clear and intentional?
5. If product-led, does it actually resemble the real product?
6. Is there any decorative junk that can be removed?
7. Does the piece feel premium and operator-grade?
8. Would this still look credible to an agency buyer or SEO lead?

If the answer to any of those is “not really,” revise.
