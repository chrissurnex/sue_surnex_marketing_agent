## 1. Purpose

This document defines the visual direction Surnex should follow so every future asset is built from one coherent system instead of random “AI startup” aesthetics.

It is grounded in:
- the current Surnex logo
- the provided lime + neutral palette
- the visual references shared by Chris
- the current product UI reality documented in `brand/SURNEX_PRODUCT_UI_REFERENCE.md`
- Surnex positioning as an AI search and SEO platform built for agencies

This is the brand’s **visual doctrine**, not a fluffy moodboard.

## 6. Design Principles

### 1. Precision beats decoration
Every visual element should either:
- clarify
- frame
- direct attention
- create brand memory

If it only decorates, it is suspect.

### 2. Contrast is a weapon
The brand lives on dark surfaces with disciplined contrast.
White and lime should hit hard because they are used intentionally.

### 3. The green is a signal, not wallpaper
The lime accent is powerful because it is selective.
Overuse turns it into noise.

### 4. Product truth matters
If we show Surnex product-inspired visuals, they must resemble the actual product DNA.
No fake floating toy dashboards.

### 5. Structure before spectacle
Strong grids, spacing, panel systems, and hierarchy matter more than effects.
Effects are last-mile enhancement, not the foundation.

### 6. Movement should suggest routing and intelligence
When motion/path visuals appear, they should imply:
- flow
- search paths
- signal routing
- decision trees
- visibility propagation

Not generic speed streaks.

## 8. Color Direction

## 8.1 Primary Accent Family
Provided palette basis:
- Lime 300: `#bbf451`
- Lime 400: `#9ae600`
- Lime 500: `#7ccf00`
- Lime 600: `#5ea500`

### Color role guidance
- **Lime 500 `#7ccf00`** = core brand accent
- **Lime 400 `#9ae600`** = highlights / glow edge / active moments
- **Lime 600 `#5ea500`** = darker state / pressed / stronger contrast on light contexts
- **Lime 300 `#bbf451`** = soft chart highlight / subtle fills / supporting highlights

## 8.2 Neutral Base Family
Provided palette basis:
- Neutral 950: `#0a0a0a`
- Neutral 900: `#171717`
- Neutral 800: `#262626`
- Neutral 700: `#404040`
- Neutral 400: `#a1a1a1`
- Neutral 100/50 for very light support only

### Neutral role guidance
- `#0a0a0a` = primary brand background
- `#171717` = primary cards/panels
- `#262626` = secondary surface
- `#404040` = dividers, strokes, subtle frames
- `#a1a1a1` = muted text
- `#fafafa` / near-white = primary text

## 8.3 Strategic Color Rule
Surnex should be overwhelmingly:
- dark
- neutral
- high-contrast

with lime used as:
- emphasis
- focus
- markers
- chart signal
- identity edge

Not as:
- giant fills everywhere
- full-page neon wash
- large generic gradients

## 9. Typography Direction

Exact font selection can come later, but the direction should be clear now.

### 9.1 Typographic personality
Use a modern, serious sans family that feels:
- technical
- premium
- readable
- product-native
- clean but not sterile

### 9.2 Directional attributes
- medium-to-tight spacing on display headlines
- bold, compact high-impact headers
- clean UI body text
- good numeric legibility
- strong support for tables, charts, and dense interface copy

### 9.3 Avoid
- overly futuristic display fonts
- rounded playful fonts
- editorial serif-led identity
- ultra-geometric fonts that reduce legibility
- quirky type treatments

### 9.4 Working type system idea
- **Display:** strong sans with authority for hero headlines
- **UI/Body:** same family or a complementary product-grade sans
- **Numbers:** clean and data-friendly

The brand should read like an operator tool, not a design magazine.

## 11. Surface System

### 11.1 Surface hierarchy
1. **Base void** — near-black brand field
2. **Primary panel** — dark elevated surface
3. **Secondary panel** — slightly lifted contrast block
4. **Micro-surface** — chart/table/inline component zones

### 11.2 Surface behavior
- restrained borders
- minimal shadow
- subtle elevation through contrast, not blur
- occasional localized glow only where it adds signal

### 11.3 Background families
Approved background systems:
- dark grid fields
- signal path flows
- modular tactical panels
- node networks
- abstract search/routing fields
- minimal dark void with precise lime accents

Not approved:
- galaxy space art
- rainbow aurora gradients
- over-detailed sci-fi cityline junk
- random network wallpaper with no structure

## 12. Product vs Marketing Relationship

This matters.

Surnex has two layers:

### Layer 1 — Real product UI
Current product is a dense dark analytics system with multicolor charting and operational dashboard structure.

### Layer 2 — Brand presentation layer
Marketing visuals can be more stylized, cleaner, and more lime-led.

### Rule
Marketing may stylize **around** the product.
It should not falsify the product.

Meaning:
- framing can be dramatic
- environment can be branded
- composition can be cleaner
- lime can shape the marketing environment

But the product itself should still look like the product.
